gnee / description

Summary

Gnee is the GUI frontend to libxnee.

With gnee you can record and replay user actions just as you can with cnee. Screenshots of gnee can be found here.

Background

Allthough Xnee is mostly used by developers (and not 'ordinary users') who are used to use command line programs we soon saw the need for a GUI program. When giving it some thoughts we realised that such a program and cnee would actually benefit from each other. As an example of the benfitial usage of the two you can develop an Xnee project file using gnee and use this with cnee when doing automatic regression tests.

You can also use Gnee to make Xnee project files that both xnee and Gnee can use. This makes it easier for people not so familiar with the X11 protocol to use Xnee (both xnee and Gnee). For more information about the project file, read the user manual.

Features

gnee comes with all features that is offered by libxnee. The same features are offered by the command line program program cnee. With gnee it is easier (than with cnee) to get started with using Xnee as a automatic test tool.
Main features:
- record an Xnee session file
- replay an Xnee session file
- create/write Xnee project files
- read Xnee project files
- distribute device events to multiple displays
