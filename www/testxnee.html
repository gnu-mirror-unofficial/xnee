Testing Xnee

Today there is no good documentation on how Xnee is tested. However all testscripts can be reached through CVS. So read the source :).

Testing Xnee
libxnee is tested through xnee,
cnee is tested automaigcally using:
scripts to test the cnee options
scripts using swinput to simulate user input in order to verify that Xnee records events correct. Swinput only works with Linux (the kernel). A picture explains a bit more how swinput is used to test xnee.
cnee is tested manually using:
gcov and gprof to search for leaks, bottlenecks etc
memprof and valgrind are used to find and track down memory leaks.
Gnee is not tested yet. Anyhow, we will probaly use Gnee to test Gnee.
For more information on how Xnee is tested, please read the Developer Manual.
